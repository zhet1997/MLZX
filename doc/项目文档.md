# 智能作文学习支持系统（Streamlit + 大模型 API + PaddleOCR）实现方案（UI细化版）

> 适用：三栏式 Streamlit 应用  
> OCR：统一采用 PaddleOCR  
> LLM：统一按现有 `llm.py` 封装调用（stream + 强制 JSON）

---

## 0. 产品定位与闭环

### 0.1 定位
本系统是“高中生议论文评改的学习支持系统”，目标是让学生在写作后**发现问题 → 明确证据 → 获得可执行处方 → 完成微任务训练 → 再评估**，形成可重复的自我提升循环。

### 0.2 闭环流程（一次作文的标准路径）
1. 左栏输入：题目 / 字数要求 / 作文（文本或图片 OCR）
2. 中栏对话：AI 追问澄清（中心论点一句话、材料要点、希望保留的亮点）
3. 右栏按钮：自评自查 / 作文点评 / 修改处方 / 微任务（生成结构化结果）
4. 中栏承载展示：所有按钮输出与对话都在中栏“历史对话展示区”沉淀
5. 学生按处方改写 / 完成微任务 → 重新触发点评/处方，形成迭代

---

## 1. UI 设计与交互规范（与界面描述对齐）

### 1.1 整体布局与风格
- **三栏式垂直布局**：左输入、中对话、右功能
- 简洁浅色基调，突出层级：输入区/对话区/操作区
- 所有“模型输出”统一进入中栏对话流，保证学习过程可追溯

---

## 2. 左侧栏：作文输入区（Input Panel）

### 2.1 输入项（垂直排列：标签 + 输入框）
1. **作文题目**（单行文本）：`title`
2. **字数要求**（单行文本）：`word_requirement`（例如“不少于800字”）
3. **作文上传**（双模式）：
   - 图片上传：JPG/PNG/JPEG（作文扫描件/照片）
   - 文本输入：直接粘贴/手动输入正文

### 2.2 OCR 交互（仅当上传图片时）
- 按钮：`OCR 识别`
- 输出：将识别结果写入“作文正文（可编辑）”文本区
- 同步展示：
  - OCR 置信度提示（例如：低置信度行列表）
  - 提醒用户手动校对（不强制）

### 2.3 分段编号（建议必做）
- 按钮：`分段编号`
- 功能：将正文按自然段切分，生成 `P1/P2/...` 段落编号
- 目的：为“证据引用（quote + para_id）”提供稳定锚点，提升点评可信度

---

## 3. 中间栏：智能交互对话区（Chat Panel）

### 3.1 顶部标题
- 固定显示：**“高中生议论文评改”**（作为产品定位提示）

### 3.2 历史对话展示区
- 按时间顺序展示多轮对话
- 对话内容来源包含两类：
  1) 学生输入（追问、补充说明）
  2) 智能体输出（包括右侧按钮触发的输出：自评报告/点评/处方/微任务）

> 规则：右侧按钮触发的结果，不单独占右栏显示；右栏只负责“触发”，结果统一进入中栏对话流。

### 3.3 底部输入框（单行）
- placeholder：“与AI的对话/AI的输出”
- 提交后触发 `chat_clarify` action：
  - AI 以“确认/选择题”为主追问
  - 生成/更新 `chat_summary`（压缩要点，减少后续 token）

---

## 4. 右侧栏：功能操作区（Toolbox Panel）

右侧栏为垂直排列的主功能区，包含四大模块（按钮 + 子按钮）。

### 4.1 自评自查（Self-check）
- 独立按钮：`自评自查`
- 交互：点击后智能体引导学生根据评分标准自评，输出**自评报告**并展示在中栏。
- 设计建议（MVP）：
  - 先用 UI 题库（选择题/单选多选）+ 规则计分得出主观“最大问题维度”
  - 再由 LLM 解释“为何判定该维度为主要短板”（可选）

### 4.2 作文点评（Commentary）
主按钮：`作文点评`  
子按钮（5个）：
- `综合`：覆盖立意/结构/论证/语言
- `立意` / `结构` / `论证` / `语言`：专项点评

输出要求（统一 JSON → 渲染为对话气泡）：
- 必须给出 `issues[]`（至少2条）
- 必须给出 `evidence[]`（引用 P段落 + 短引 + why）
- `diagnosis.main_problem_dimension` 用于提示学生本次优先修哪一块

### 4.3 修改处方（Prescription）
主按钮：`修改处方`  
子按钮（5个）：
- `综合` / `立意` / `结构` / `论证` / `语言`

输出要求：
- 强可执行：`next_steps[]` 每条必须是“能照做的动作”，避免空泛建议
- 建议 3–5 条为宜
- 对应引用证据（说明为什么要改这处）

### 4.4 微任务（Microtasks）
主按钮：`微任务`  
子按钮（4个）：
- `立意` / `结构` / `论证` / `语言`

输出要求：
- 每个微任务强调“10分钟能做完”
- 必须包含自检规则（check_rule），让学生自己判断是否完成
- 可选给一个示例（但要短）

## 5. 技术架构与模块划分

### 5.1 总体架构（单体应用 + 模块化）
- Streamlit：UI + session_state 状态管理
- core/actions.py：按钮路由（action → prompts → llm.py）
- core/ocr：PaddleOCR 识别与后处理
- core/schemas.py：Pydantic 定义 JSON 输出结构（防止 UI 崩）
- core/paragraphing.py：分段、编号、证据引用定位
- core/caching.py：缓存（避免重复 OCR/重复调用 LLM）
