# 改动文档：微任务训练（可选题 → 计时作答 → 自动提交 → 10分制评分）

## 0. 目标
在“微任务”功能中新增“计时作答与评分”模式：
- 选择某一维度微任务（立意/结构/论证/语言）后，仍然由系统生成 4 个题目（保持现有逻辑）。
- 这 4 个题目在中间对话区以“可点击选项”形式呈现，提示学生选择 1 个题目。
- 学生点击某题后进入“专注作答模式”（modal）：页面其余区域灰化不可操作，显示倒计时。
- 学生在输入框里作答，点击提交；若倒计时结束则自动提交。
- 提交后调用大模型 API（按既有 `llm.py`）结合预制 prompt，输出 10 分制评分 + 简短点评，并回填到中间对话区。

---

## 1. UX / UI 行为规范（与现有三栏一致）

### 1.1 触发入口
右栏：微任务 → 选择维度按钮（立意/结构/论证/语言）
- 行为：生成 4 个微任务题目（不变）

### 1.2 中栏展示：题目列表（可交互）
生成后，中栏对话气泡新增一个“题目选择块”：
- 标题：`请选择一个微任务题目开始训练（限时）`
- 展示形式：4 个可点击选项（建议用按钮或 radio）
- 每个题目显示：
  - 题目标题（短）
  - 可选：题目说明（可折叠/小字）

点击某个题目后进入“专注作答模式”。

### 1.3 专注作答模式（Modal Overlay + 灰化）
进入模式后：
- 页面其他区域灰化，并禁用交互（左栏输入、右栏按钮都不可点击）
- 中栏出现一个“浮层对话框”：
  - 顶部：倒计时（例如 10:00）
  - 中部：题目全文
  - 底部：多行输入框（学生作答）
  - 右侧/底部：提交按钮
- 计时结束：自动触发提交（如果学生已输入则提交；若空则提交空答案并提示）

提交完成后退出专注模式，恢复页面交互，并在中栏追加评分结果气泡。

## 2. 状态机设计（Streamlit 可实现）

### 2.1 核心状态（存入 st.session_state）
新增以下 session_state 字段：

- `microtask_pool`: dict
  - 保存最近一次生成的 4 题（含维度、题目列表、生成时间、版本等）
- `microtask_selection`: dict | None
  - 当前选中的题目（id、题面、维度）
- `focus_mode`: bool
  - 是否处于专注作答模式（overlay 开启）
- `focus_deadline_ts`: float | None
  - 该题倒计时结束的绝对时间戳（time.time() + duration）
- `focus_duration_sec`: int
  - 默认时长（例如 600 秒，可配置）
- `focus_answer_text`: str
  - 当前作答内容（输入框绑定）
- `focus_submitted`: bool
  - 防重复提交（自动提交与手动提交可能同时触发）
- `microtask_grade_result`: dict | None
  - 最近一次评分结果（10分制 + 简评 + 改进建议）
- `ui_lock`: bool
  - 页面锁（灰化禁用），通常与 focus_mode 同步

### 2.2 状态流转
1) 点击右栏“微任务-维度”按钮：
- 调用 action：`microtask_dim`（原有）
- 返回 4 道题（JSON），写入 `microtask_pool`
- 在中栏渲染“题目选择块”

2) 点击某一道题（题目选择块）：
- 写入 `microtask_selection`
- `focus_mode=True`, `ui_lock=True`
- 设置 `focus_deadline_ts = now + duration`
- `focus_submitted=False`
- 清空/初始化 `focus_answer_text`

3) 手动提交 or 倒计时结束自动提交：
- 若 `focus_submitted=True`：直接 return（防抖）
- 否则：
  - `focus_submitted=True`
  - 调用评分 action：`microtask_grade`
  - 将结果追加进 chat_history（中栏）
  - 退出 focus：
    - `focus_mode=False`, `ui_lock=False`
    - 清空 selection/answer/deadline（可选保留池）

## 3. LLM 动作新增：microtask_grade（评分）

### 3.1 新增 Action
新增 action 名：`microtask_grade`

输入（user 内容）应包含：
- 作文题目（title）
- 字数要求（word_requirement）
- 作文原文（可选；建议只传摘要或段落编号片段，避免 token 爆炸）
- 微任务维度（dimension）
- 被选中的微任务题面（task_text）
- 学生回答（answer_text）
- 评分规则：10 分制 + 简评（需预制 prompt 中明确）

输出：严格 JSON（用于 UI 渲染、缓存、导出）

### 3.2 JSON Schema（建议）
```json
{
  "meta": {"action":"microtask_grade","version":"1.0"},
  "score": 0,
  "comment": "一句话点评（简短）",
  "strengths": ["..."],
  "issues": ["..."],
  "next_steps": ["..."],
  "rubric": {
    "dimension": "立意|结构|论证|语言",
    "criteria": ["..."]
  }
}
